# codex

Небольшой пример дашборда с источниками JSON-данных.

## Что реализовано

- Каждый JSON в `data/` содержит метаданные: `updated_at`, `source`, `status`, `errors`.
- Фронтенд (`frontend/`) показывает:
  - timestamp последнего обновления по каждому источнику;
  - глобальный индикатор «данные могут быть устаревшими»;
  - понятные empty-state состояния, если источник пустой или недоступен.
- Скрипт health-check `scripts/validate-data.mjs` валидирует схему JSON.

## Проверка данных перед коммитом

```bash
npm run validate:data
```

Также добавлен `precommit`-скрипт:

```bash
npm run precommit
```

## Локальный просмотр фронтенда

Запустите любой статический сервер из корня репозитория, например:

```bash
python3 -m http.server 4173
```

После этого откройте:

- `http://localhost:4173/frontend/`
